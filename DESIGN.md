# FyMail - 邮件营销系统设计文档

## 项目概述
**项目名称**: FyMail (mail.fywarehouse.com)
**业务定位**: FYWarehouse LCL拼箱到加拿大服务的拓客邮件系统
**目标用户**: 欧洲货代公司（德国、荷兰、英国、法国、比利时）
**核心价值**: 通过批量模板邮件开发欧洲货代潜在客户

## 业务需求

### 核心功能需求
1. **邮件模板管理系统**
   - 富文本邮件编辑器
   - 变量替换系统（{公司名}, {联系人}, {国家}等）
   - 多语言模板支持（中/英/法/德语）
   - A/B测试模板对比

2. **欧洲货代联系人管理**
   - CSV/Excel批量导入
   - 按国家/业务类型分组标签
   - 邮箱验证与去重
   - 联系人搜索与筛选

3. **批量邮件发送引擎**
   - 发送队列管理
   - 智能发送间隔控制（防垃圾邮件）
   - 发送进度实时监控
   - 失败重试机制

4. **发送统计与分析**
   - 发送成功率统计
   - 打开率/点击率追踪（HTML邮件）
   - 回复率监控
   - 数据可视化图表

5. **数据采集集成**
   - 欧洲货代公司自动搜索
   - 联系人信息提取
   - 数据清洗与验证

### 非功能性需求
- **性能**: 支持每分钟发送50+封邮件
- **可靠性**: 发送成功率>95%
- **安全性**: 防止滥用，数据加密
- **可用性**: 7x24小时服务，99.9%可用性
- **扩展性**: 支持未来功能扩展（短信、社交媒体集成）

## 技术架构

### 整体架构
```
用户浏览器 → Cloudflare CDN → Nginx反向代理 → Next.js应用 → PostgreSQL数据库
                                              ↓
                                        邮件发送队列 (Bull + Redis)
                                              ↓
                                        邮件发送引擎 (Nodemailer)
                                              ↓
                                        Gmail SMTP (ops@fywarehouse.com)
```

### 技术栈选择
- **前端**: Next.js 14 (App Router) + TypeScript + Tailwind CSS + Shadcn/ui
- **后端**: Next.js API Routes + Prisma ORM
- **数据库**: PostgreSQL (生产) / SQLite (开发)
- **邮件引擎**: Nodemailer + Nodemailer Express Handlebars
- **队列系统**: Bull (Redis) 用于异步任务
- **部署**: 现有VPS (43.166.132.131) + PM2进程管理
- **域名**: mail.fywarehouse.com (Cloudflare管理)

## API接口设计

### 核心API端点
1. **邮件模板API**
   - GET /api/templates - 获取模板列表
   - POST /api/templates - 创建新模板
   - PUT /api/templates/:id - 更新模板
   - DELETE /api/templates/:id - 删除模板
   - POST /api/templates/:id/preview - 预览模板

2. **联系人API**
   - GET /api/contacts - 获取联系人列表
   - POST /api/contacts - 创建联系人
   - POST /api/contacts/import - 批量导入
   - PUT /api/contacts/:id - 更新联系人
   - DELETE /api/contacts/:id - 删除联系人

3. **发送活动API**
   - GET /api/campaigns - 获取活动列表
   - POST /api/campaigns - 创建新活动
   - POST /api/campaigns/:id/send - 开始发送
   - GET /api/campaigns/:id/stats - 获取统计

4. **邮件发送API**
   - POST /api/mail/send-test - 发送测试邮件
   - POST /api/mail/send-bulk - 批量发送

## 风险评估与缓解

### 技术风险
1. **发送被标记为垃圾邮件**
   - 缓解：使用专业SMTP服务、遵守反垃圾邮件规则、逐步增加发送量

2. **Gmail发送限制**
   - 缓解：监控发送配额、多账号轮换、考虑专业邮件服务

3. **数据库性能问题**
   - 缓解：合理索引设计、查询优化、定期维护

### 业务风险
1. **数据质量问题**
   - 缓解：邮箱验证、数据清洗、人工审核

2. **合规性问题**
   - 缓解：遵守GDPR等法规、提供退订机制、数据隐私保护
