## 三、架构与网络拓扑

### 3.1 AI 对话功能链路
```
浏览器 → mail.fywarehouse.com (CF Worker)
       → /api/chat → Worker fetch
       → clawcc.fywarehouse.com/api/chat-exec
       → nginx → localhost:3000 (Next.js)
       → claude CLI (子进程)
```

### 3.2 DNS 与代理
- `mail.fywarehouse.com` — Cloudflare Worker（前端 + API 代理）
- `clawcc.fywarehouse.com` — nginx 反代到本机 Next.js
- `origin-mail.fywarehouse.com` — DNS-only 记录指向 43.166.132.131
- nginx 配置了 `^~ /api/chat-exec` 优先级路由规则

### 3.3 项目间关系
```
fywarehouse-nextjs (官网前端)
    ├── 内嵌 FyMail 营销工作台 UI
    └── 调用 Skills API

fymail (独立后台)
    ├── 独立的邮件营销管理
    ├── AI 对话功能（/chat → claude CLI）
    └── 依赖 fymail-intelligence 做市场情报

fymail-intelligence (API 网关)
    └── 为 fymail 提供数据支撑

OpenClaw (AI 助手)
    └── workspace/ 下存放 fywarehouse-nextjs
```

## 五、关键技术决策与约束

1. **Cloudflare Worker 不能执行子进程** — 所以 chat API 必须回源到服务器
2. **CSS Modules 而非 Tailwind** — fywarehouse 官网的样式方案
3. **Mock 模式** — fymail-intelligence 支持无 Redis/ES 运行，便于开发
4. **技能系统** — fywarehouse 有完整的技能注册/发现/执行/监控体系
